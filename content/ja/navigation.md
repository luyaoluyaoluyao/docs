## 1つの紹介

Mendix Studioの **Navigation Document** は、ツリー形式でアプリケーションの構成されたメニューを表示します。 ナビゲーションに項目とサブ項目を作成できます。

![](attachments/navigation/navigation-vs-app.png)

**ナビゲーション ドキュメント**を開くには、左のメニューバーの対応するアイコンをクリックします。

![ナビゲーションドキュメントアイコン](attachments/navigation/navigation-icon.png)

**Navigation Document** は、エンドユーザーがアプリをナビゲートしたり、特定のアクションを実行したりできるメニュー項目で構成されています。 たとえば、メニュー項目を設定して、特定のページを開いたり、エンドユーザーをプロファイルからログアウトさせることができます。 メニューアイテムに割り当てることができるアクションの詳細については、 [イベント セクション](#events-section-navigation) を参照してください。

サブアイテムをメニューアイテムに追加することもできます。 サブアイテムを持つメニューアイテムにアクションを割り当てることはできません。

新しいメニュー項目を作成するには、次の操作を行います。

1. 左メニューバーの **ナビゲーション ドキュメント** アイコンをクリックして、 **ナビゲーション** を開きます。

2. ナビゲーションツリーの下部にあるプラスをクリックしてメニュー項目を作成します または、既存のナビゲーション項目の横にあるプラスをクリックしてサブ項目を作成します

   ![](attachments/navigation/adding-navigation-items.png)

3. 必要に応じて作成されたアイテムのプロパティを指定します (詳細については、 [メニュー アイテムのプロパティ](#properties-of-menu-items) を参照してください)。

新しいメニューアイテムまたはサブアイテムがナビゲーションに追加されます。

{{% alert type="info" %}}

Studio では、ナビゲーションプロファイルのレスポンシブタイプを表示および編集し、Studio Pro ではプロファイルの種類が他にもあります。 Studio Pro のプロファイルの詳細についてはこちら [Studio Pro Guide](/refguide/navigation#profiles) の *ナビゲーション* の *プロファイル* セクションを参照してください。

{{% /alert %}}

アプリケーションでナビゲーションを構成する方法の詳細については、 [ナビゲーションバーを構成する方法](/studio-how-to/navigation-how-to-configure) を参照してください。

## 2つのナビゲーションエディターのプロパティ

ナビゲーションエディタのプロパティでは、 [デフォルトのホーム ページ](#default-home-page) と [ロール固有のホーム ページ](#role-specific-home-page) を設定できます。

### 2.1 デフォルトのホームページ {#default-home-page}

ページまたはマイクロフローをデフォルトのホームページとして設定できます。

![デフォルトのホームページ](attachments/navigation/default-home-page.jpg)

**ホームページ** セクションは以下のプロパティで構成されています。

* **Page** – エンドユーザーがアプリを開いたときに開かれるページを設定できます。
* **Microflow** – エンドユーザーがアプリを開いたときに実行されるマイクロフローを設定できます。 マイクロフローは、いくつかのロジックを実行したり、ホームページに渡すべき特定のデータを収集するために使用することができます。

ホームページアイコンは、ホームページとして設定されているページまたはマイクロフローに対してページまたはマイクロフローのリストに表示されます。

![ホームアイコン](attachments/navigation/home-icon.jpg)

### 2.2 役割別ホームページ {#role-specific-home-page}

ロール固有のホームページは、 [デフォルトのホームページ](#default-home-page) を上書きし、指定されたロールを持つユーザに対してのみ開かれるホームページです。 アプリケーションの各ロールごとに別のホームページを設定できます。 ロールとそれらを作成する方法の詳細については、 [セキュリティ、ロール & 権限](settings-security) を参照してください。

例えば、管理者ロールでは、 **Admin_Dashboard** ページをホームページとして設定できます。 一方、他のユーザーはアプリを開いたときにデフォルトのホームページを開きます。

![役割別ホームページ](attachments/navigation/role-specific-home-page.jpg)

ロール固有のホームページを追加または編集する場合は、次のプロパティを設定できます。

* **ロール** – 専用のホームページを持つロールを定義します。

* **Type** – ユーザーがアプリを開いたときにページまたはマイクロフローを開くかどうかを定義します。 次のタイプが使用できます:

  * **ページ** – ページをホームページとして設定します。
  * **Microflow** – マイクロフローがホームページとして設定されます。すなわち、ユーザーがアプリを開いたときにマイクロフローが実行されます。 マイクロフローは、いくつかのロジックを実行したり、ホームページに渡すべき特定のデータを収集するために使用することができます。

* **ページ** – このプロパティは、 **タイプ** が **ページ** に設定されている場合にのみ使用できます。 選択したロールに対して開かれた専用のホームページを設定できます。

* **Microflow** – このプロパティは、 **Type** が **Microflow** に設定されている場合にのみ使用できます。 選択したロールを持つユーザーがアプリを開いたときに実行されるマイクロフローを設定できます。

    ![](attachments/navigation/role-specific-home-page-properties.jpg)

ロール固有のホームページアイコンは、ホームページとして設定されているページまたはマイクロフローに対してページまたはマイクロフローのリストに表示されます。

![役割固有のホームページアイコン](attachments/navigation/role-specific-home-page-icon.jpg)

### 2.3 役職固有のホームページの追加

ロール固有のホームページを追加するには、次の操作を行います。

1. [セキュリティ](settings-security) が有効になっていることを確認してください。

2. ナビゲーション エディター > ナビゲーション エディターのプロパティを開きます。

3. **Role-Specific Home Page** セクションで、 **Add Role-Specific Home Page** をクリックします。

4. **ロール** ドロップダウン メニューで、専用のホーム ページを持つロールを選択します。

    ![ロールの選択](attachments/navigation/selecting-role.jpg)

5. ホームページの **タイプ** を選択します。

6. 選択した種類に応じて。 選択したロールを持つユーザーがアプリケーションを開いたときに開かれるページまたはマイクロフローを選択します。

ロール固有のホームページが設定されています。

## 3 メニューアイテムのプロパティ {#properties-of-menu-items}

メニューアイテムのプロパティは以下のセクションで構成されています:

* [イベント](#events-section-navigation)
* [全般](#general-section-navigation)

![](attachments/navigation/navigation-properties.png)

### 3.1 イベントセクション {#events-section-navigation}

**イベント** セクションで **クリックアクション** を選択できます。 **on Click Action** は、ユーザーがメニューアイテムをクリックしたときに実行されるアクションを定義します。

{{% alert type="info" %}}
メニュー項目にサブ項目がある場合、その項目に対して設定されたアクションを持つことはできません。
{{% /alert %}}

使用可能なアクションは以下の表に記載されています:

| アクション    | 説明                                                                                                                                                                                                                                                                                                  |
| -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| なし       | アクションは実行されません。                                                                                                                                                                                                                                                                                      |
| ページ      | 指定されたページが開きます。                                                                                                                                                                                                                                                                                      |
| マイクロフロー  | 選択したマイクロフローが実行されます。                                                                                                                                                                                                                                                                                 |
| 変更を保存    | 現在開いているページに加えられたすべての変更を保存(コミット)し、ページを閉じます。                                                                                                                                                                                                                                                          |
| 変更をキャンセル | 現在開いているページに加えられたすべての変更をロールバックし、ページを閉じます。                                                                                                                                                                                                                                                            |
| ページを閉じる  | ポップアップウィンドウ(ポップアップページ用)を閉じるか、以前に訪れたページに移動します。 このアクションはページを閉じ、変更は保存されない場合には閉じることに注意してください。 **変更を保存** をこの目的に使用します。                                                                                                                                                                                    |
| サインアウト   | 現在のユーザーはアプリからサインアウトされています。                                                                                                                                                                                                                                                                          |
| リンクを開く   | リンクタイプに基づいてアクションをトリガーします: <ul><li>**Web** – ウェブサイトにナビゲートします </li><li>**メール** – メールを作成</li><li>**電話** – 通話を開始</li><li>**テキストメッセージ** - テキストメッセージを送信</li></ul>**Note** When you configure **Email**, **Phone Call** or **Message** options, the corresponding default app will be opened on the device when the action is triggered, for example, the default email client will be opened to compose a message. |

{{% alert type="info" %}}

メニュー項目にサブ項目がある場合、 **クリックアクション** は **何も**でなければなりません。

{{% /alert %}}

### 3.2 全般 {#general-section-navigation}

**一般** セクションで設定可能なプロパティは次のとおりです。

* **図表番号** – メニュー項目の名前を入力できます。
* **アイコン** – メニュー項目のアイコンを設定できます。
