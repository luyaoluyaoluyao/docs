参照セットセレクターでは、オブジェクトを選択することで、参照セットの関連付けを設定できます。 例えば、顧客が複数のグループに属することができる場合、参照セットセレクタを使用して顧客が属するグループを選択できます。 これには、ドメインモデルに設定されているタイプ参照のグループへの顧客からの関連付けが必要です。

{{% alert type="info" %}}

![](attachments/pages/reference-set-selector.png) 顧客に関するこのデータビューでは、参照セットセレクタを使用して複数のグループを選択できます。

![](attachments/pages/reference-set-selector-domain-model.png) ドメインモデルには、顧客からグループへの参照セットの関連付けがあります。

{{% /alert %}}

リファレンスセットセレクタは [データ グリッド](data-grid) のように見えるため、多くのプロパティをデータ グリッドと共有します。 主な違いは、参照セットセレクタに検索バーがなく、デフォルトで異なるボタンがあることです (Add, Remove)。

{{% alert type="warning" %}}

**追加** と **削除** ボタンは、データ グリッドでデフォルトで使用される **新規** および **削除** ボタン以外の何かを意味します:

*   **追加** ボタンは、既存のオブジェクトへの参照を追加します。 一方、 **Create** ボタンは、新しいオブジェクトを作成し、参照を追加しません。
*   The **Delete** button actually deletes an object for good, whereas the **Remove** button simply removes the reference to the object and leaves the object itself intact.

{{% /alert %}}

## 外観のプロパティ

### コントロールバーを表示

このプロパティは、エンドユーザーインターフェイスでコントロール バーが表示されるかどうかを示します。 コントロールバーにはページングボタンも含まれています。

{{% alert type="warning" %}}

コントロールバーが非表示の場合でも、行をクリックすることによってトリガーされる既定のボタンが残っている場合があります。 詳細については、プロパティ 'Default button trigger' と [control bar](control-bar) を参照してください。

{{% /alert %}}

_デフォルト値:_ True

### ページングボタンを表示

このプロパティは、グリッド内の情報をページに移動するボタンが表示されているかどうかを示します。 グリッドの行数より多くのオブジェクトが存在しないことを確認している場合にのみ、これらのボタンを非表示にします。 コントロールバーを非表示にすると、ページングボタンも非表示になります。

_デフォルト値:_ True

### 列の重み付け

列の重みは、半コロンで区切られたパーセンテージです。 列の幅を決定します。 重量は最大100\まで加算する必要があります。 列の幅を変更する別の方法は、列間の区切り線をドラッグすることです。

{{% alert type="info" %}}

上のスクリーンショットでは、列の重みは 63;37 です。

{{% /alert %}}

### 行数

このプロパティを使用すると、1ページに表示される行数を変更できます。 プロパティ「空の行を表示」も参照してください。

_デフォルト値:_ 5

### 空の行を表示

空の行を表示する場合、ページに表示するオブジェクトが少なくても、同じ行数を表示するグリッドが常に存在します ('行数' を参照)。

_デフォルト値:_ False

## ビヘイビアのプロパティ

### 選択モード

選択モードでは、ユーザーがグリッド内の項目を選択できるかどうかと方法を決定します。

| 値         | 説明                                                                                                      |
| --------- | ------------------------------------------------------------------------------------------------------- |
| 選択されていません | ユーザーはアイテムを選択できません。                                                                                      |
| 単一選択      | ユーザーはそれをクリックして単一の項目を選択することができます。 別の項目をクリックすると、その項目が選択されます。                                              |
| 複数選択      | 最初の項目をクリックし、「Ctrl」キーを押しながら他の項目をクリックすることで複数項目を選択できます。 単にアイテムをクリックすると、すべてのアイテムの選択が解除され、クリックしたアイテムが選択されます。 |
| 単純な複数選択   | ユーザーは、単にそれらをクリックすることによって複数の項目を選択することができます。 これは、バージョン 3.1.0 以前のデフォルトの複数選択動作でした。                          |

_デフォルト値:_ 単一選択

### 既定のボタントリガー

既定のボタンは、1行またはダブルクリックでトリガーできます。

| 値            | 説明                                                            |
| ------------ | ------------------------------------------------------------- |
| シングルクリック     | ワンクリックで既定のボタンがトリガーされます。 これはユーザーが行を選択できるようにするために組み合わせて使用できません。 |
| Double click | ダブルクリックするとデフォルトのボタンがトリガーされます。                                 |

_デフォルト値:_ ダブルクリック

### ツールチップページ

ツールチップページは、行の上にマウスをポイントしたときに表示されるページです。 ツールチップページはデータ グリッドと同じエンティティのデータ ビューで構成する必要があります。 作成と接続の上に、ツールチップページが表示される列を指定する必要があります。 [データ グリッド 列](columns) のプロパティ 'ツールチップを表示' を参照してください。

## データソースのプロパティ

### エンティティパス

参照セットセレクターのエンティティパスは、その関連付けの反対側のエンティティに含まれているデータビューのエンティティから 'reference set' 型の関連付けに続くパスです。 上の例では、顧客からグループへの関連 Customer_Group がフォローされ、パスは次のとおりです。

{{% alert type="warning" %}}

参照セレクタでは、タイプ参照集合(Customer_Group)の関連付けを編集します。 セレクターを使用したオブジェクトの追加と削除は、それらのオブジェクトへの参照のみを追加および削除します。オブジェクト自体は変更されません。

{{% /alert %}}

{{% alert type="warning" %}}

参照セットセレクタで非持続可能エンティティ (NPE) を使用することはまだサポートされていません。 そのため、参照セットセレクタのエンティティセレクタで NPE を選択することはできません。

{{% /alert %}}
