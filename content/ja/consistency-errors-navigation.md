## 1つの紹介

このドキュメントでは、Mendix Studio でナビゲーションを構成する際に発生する最も一般的な一貫性エラーを解決する方法を説明します。 ナビゲーションの詳細については、 [Navigation Document](navigation) を参照してください。

一貫性エラーの例として、データビューをメニュー項目として設定したページがあります。

### 2.1 選択されたページがオブジェクトを期待する場合のエラー修正 {#page-expects-an-object}

データビューをメニュー項目として設定すると、一貫性エラーが表示されます。 なぜなら、ページはオブジェクトが渡されることを期待しているからです。

たとえば、 **Program** というメニュー項目が作成されています。 このメニュー項目は **プログラム** ページを開きますが、オブジェクトを作成して渡すことはありません。 **プログラム** ページにはデータビューがあり、 *ProgramItem* オブジェクトが渡されることを期待しています。 その結果、ナビゲーションからこのページにオブジェクトが渡されないので、一貫性エラーが発生します。

![メニューアイテムエラーを表示するスキーム](attachments/consistency-errors-navigation/page-expects-an-object-error.png)

エラーを修正するには、オブジェクトを作成してページに渡すことができます。 次の操作を行います:

1. ナビゲーション ドキュメントを開きます。

2. **プログラム** メニュー項目のプロパティを開き、次の操作を行います:<br />

    a In the **Events** section, make sure that the on-click action is set to **Page** and enable the **Create Object** option.<br />

    B **ProgramItem** を **エンティティ** に設定します。<br />

    {{% image_container width="350" %}}![メニューアイテムのプロパティ](attachments/consistency-errors-navigation/menu-item-properties.png)
    {{% /image_container %}}

今、ユーザーがナビゲーション項目をクリックしたとき。 *ProgramItem* 型の新しいオブジェクトが作成され、 **Program** ページに渡されます。

### 2.2. ホームページがオブジェクトを期待するときのエラー修正 {#home-page-expects-an-object}

オブジェクトがホームページとして渡されることを期待するページを設定した場合 (例えば、 データビューのページ)、一貫性のエラーが表示されます。

例えば、 **ナビゲーション**では、 **顧客詳細** ページをホームページとして設定しています。

![ナビゲーションアイテムのプロパティ](attachments/consistency-errors-navigation/navigation-item-properties.png)

**顧客詳細** ページには、オブジェクト *顧客* が渡されることを期待するデータ ビューがあります。 言い換えれば、このページはデータを最初に表示できるようにする必要があります。

{{% image_container width="350" %}}![データビューが顧客オブジェクトを期待する](attachments/consistency-errors-pages/data-view-customer.png)
{{% /image_container %}}

ただし、ホームページはデフォルトでアプリで開かれる最初のページです。 と、最初にデータを取得する必要があるため、 **Customer Details** ページはホームページとして機能しません。

この例では、 エラーを解決する最善の方法は、別のページをデータビューがないホームページとして設定することです。 メニュー項目のナビゲーションとプロパティの詳細については、 [ナビゲーション ドキュメント](navigation) を参照してください。

ただし、 新しい *顧客* オブジェクトを作成し、ページに渡すマイクロフローを作成することで、このエラーを解決することもできます。 次の操作を行います。

1. ホームページメニュー項目の **ナビゲーション** > プロパティを開きます。

2.  **On Click Action** from **Page** to **Microflow**.

    {{% image_container width="350" %}}![メニューアイテムのプロパティ](attachments/consistency-errors-navigation/menu-item-on-click-action.png)
    {{% /image_container %}}

3. **Select Microflow** をクリックし、ポップアップウィンドウで **New Microflow** をクリックします。

4. マイクロフローに名前を付けます *ACT_Open_HomePage*.

5. 作成されたマイクロフローが開きます。 **Toolbox** > **Object Activities**で、 **Create Object**を選択し、それをマイクロフローにドラッグ&ドロップします。

6.  **Create object** activity properties, and set **Entity** to **Customer**.

    {{% image_container width="350" %}}![オブジェクトのアクティビティプロパティを作成](attachments/consistency-errors-navigation/create-object-properties.png)
    {{% /image_container %}}

7. **Toolbox** > **Client Activities**で、 **Show Page** activityを選択し、それをマイクロフローにドラッグ&ドロップします。

8.  **ページの表示** アクティビティプロパティを開き、次の操作を行います:<br />

    a  **ページ** を **ホーム** に設定します。<br />

    B **オブジェクトを** に **新規顧客**に渡すように設定します。<br />

    {{% image_container width="350" %}}![ページのプロパティを表示](attachments/consistency-errors-navigation/show-page-properties.png)
    {{% /image_container %}}

これで、 *Customer* 型の新しいオブジェクトが作成され、ホームページに渡されます。

![ホームページマイクロフローを開く](attachments/consistency-errors-navigation/open-page-microflow.png)
