## 1つの紹介

このドキュメントでは、Mendix Studio でページを構成する最も一般的な一貫性エラーを解決する方法を説明します。

整合性エラーの例として、ページ上のデータビューのentityプロパティを指定しない場合があります。

## 2つのリストビュー一貫性エラー

[リスト ビュー](page-editor-data-view-list-view#list-view-properties) のデータ ソースを正しく構成しないと、一貫性エラーが発生します。

以下のスキームは、リスト ビューのデータ ソースが **データベース**に設定されていることを示しています。 しかし、データベースから取得する必要がある特定のエンティティは設定されていません。 これにより一貫性エラーが発生します。![リスト表示エラー 原因](attachments/consistency-errors-pages/list-view-error.png)

以下の表は、リストビューの設定時に発生する最も一般的なエラーについて説明しています。 これらのエラーの原因とそれらの修正方法。

| エラーコード | チェックパネルのテキスト                                                         | エラーの原因                                                                                                                                                                          | 修理方法                                                                                          |
| ------ | -------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| CE0488 | このリスト ビューのデータ ソースに対して設定されたエンティティがありません。 エンティティを選択するか、データソースを変更します。   | The **Database** (or **XPath**/**Association**) option is selected as the data source for the list view on the page has been set, but the **Entity** property is not specified. | リスト ビューのプロパティ > **データ ソース** セクションを開き、 **エンティティ** フィールドでエンティティを選択します。                          |
|        | このリスト ビューのデータ ソースにマイクロフローが設定されていません。 マイクロフローを選択するか、データソースを変更します。     | データ ソースが **Microflow**に設定されていますが、特定のマイクロフローは選択されていません。                                                                                                                          | リスト ビューのプロパティ > **データ ソース** セクションを開き、 **Microflow** フィールドでマイクロフローを選択します。 または、データソースの種類を変更します。 |
|        | このリスト ビューのデータ ソースに設定されている nanoflow はありません。 ナノフローを選択するか、データソースを変更します。 | データソースとしてnanoflowが選択されていますが、特定のnanoflowは選択されていません。                                                                                                                              | Studio Pro を開いて、このリスト ビューでナノフローを選択するか、データ ソースの種類を変更します。                                       |

適切な方法で [データ ビュー](page-editor-data-view-list-view#data-view-properties) のデータ ソースを構成しないと、一貫性エラーが発生します。 たとえば、データ ソースとしてリスト ウィジェットを選択しました。 しかし、あなたは特定のリストを選択していません。

|        | このデータ ビューのデータ ソースに設定されているマイクロフローはありません。 マイクロフローを選択するか、データソースを変更します。   | データ ソースが **Microflow**に設定されていますが、特定のマイクロフローは選択されていません。                 | データ ビューのプロパティ> **データ ソース** セクションを開き、 **Microflow** フィールドでマイクロフローを選択します。 または、データソースの種類を変更します。 |

### 4.1 エラー修正例 1 {#error-example-1}

ページが別のページやマイクロフローから渡されないコンテキストを期待すると、一貫性エラーが発生します。 

Let us study an example: the **Customers** page contains a list view with a list of all customer names (**Customer** is set as **Entity** in the **Data Source** properties), and a **Details** button outside of the list view (placed in a [container](page-editor-widgets-structure#container-overview) only). **詳細** ボタンは、ユーザーがクリックすると **顧客詳細** ページを開きます ( **クリック時** ボタンが **ページ** に設定されており、 **オブジェクトの作成** オプションが無効になっています)。 

![顧客ページ上のボタンのプロパティ](attachments/consistency-errors-pages/customers-page.png)

ただし、 **Customer Details** ページには、オブジェクト *Customer* が渡されることを期待するデータ ビューがあります。 言い換えれば、このページはデータを最初に表示できるようにする必要があります。 

{{% image_container width="350" %}}![データビューが顧客オブジェクトを期待する](attachments/consistency-errors-pages/data-view-customer.png) 

{{% /image_container %}}

このオブジェクトは **顧客** ページから渡されていないため、一貫性エラーが発生します。

![コンテキストが利用できない場合のエラーの例](attachments/consistency-errors-pages/object-error.png)

**顧客** ページの **詳細** ボタンは、データコンテナ（データビューまたはリストビュー）の外側にあるため、どの顧客 (オブジェクト) を通過すべきかは分かりません。 このエラーを修正する方法は、以下によって異なります:

* You want to pass a specific *Customer* object to the **Customer Details** page, in other words, the details of a particular customer will be displayed on the **Customer Details** page (for more information, see section [4.1.1 Passing a Specific Object to the Page](#passing-specific-object))
* You want to create a new object of the *Customer* type and pass it to the **Customer Details** page, this means that a new customer will be created (for more information, see section [4.1.2 Creating a New Object and Passing it to the Page](#creating-new-object))



#### 4.1.1 特定のオブジェクトをページに渡す {#passing-specific-object}

特定の顧客の詳細を開くために **顧客詳細** ページをご希望の場合 つまり、特定のオブジェクトをページに渡したいということです。 すでに **顧客** ページに顧客リストが表示されているリスト表示があるため、次の方法でこのエラーを修正できます。

1. **顧客** ページを開きます。

2.  リスト ビュー内の **詳細** ボタンをドラッグします。
   
   {{% image_container width="350" %}}![リストビューの例](attachments/consistency-errors-pages/list-view-content.png) 
   
   {{% /image_container %}}

Now the button gets the object of type *Customer* from the list view on the **Customers** page, and it will be passed to the **Customer Details** page. その結果、特定の顧客の詳細が **顧客詳細** ページに表示されます。 



#### 4.1.2 新しいオブジェクトを作成し、ページに渡す。 {#creating-new-object}

新規顧客を作成し、 **顧客詳細** ページに顧客の詳細を記入する場合。 次のことができます。

1. **顧客** ページを開きます。

2. **詳細** ボタン > **イベント** セクションのプロパティを開き、 **オブジェクトの作成** オプションを有効にします。
   
   {{% image_container width="350" %}}![オブジェクトの作成オプションが有効になっています](attachments/consistency-errors-pages/create-object-option-enabled.png) 
   
   {{% /image_container %}}

3. **顧客** を **エンティティ** として設定します。

4. ボタンのキャプションを **詳細** から **追加**に変更する このボタンは新規顧客を作成し、既存の顧客の詳細を表示しません。

ユーザーがこのボタンをクリックすると、 **Customer Details** ページが開き、新しい *Customer* オブジェクトが作成されます。 



### 4.2 エラー修正例 2 {#error-example-2}

ウィジェットがページを開き、このウィジェットがエンティティ X のデータ コンテナ内にある場合。 しかし、参照ページはエンティティYを期待しており、一貫性エラーが発生します。 

例えば、 **タスク** ページを開く **エンジニア** ページに **新しい** ボタンがあります。 

{{% image_container width="350" %}}![エンジニアページのボタン](attachments/consistency-errors-pages/engineers-page.png) 

{{% /image_container %}}

The button is placed inside a list view; the list view's data source is set to entity *Engineer* in **Properties** > **Data Source**.

The Tasks page has a data view on it, but the data view's data source is set to entity *SmartTask* in **Properties** > **Data Source**. 

This means that data view expects the object of type *SmartTask* passed to it, but the **Engineers** page is passing the object of type *Engineer*. 結果として一貫性エラーが発生します。

{{% image_container width="350" %}}![タスク ページの一覧表示](attachments/consistency-errors-pages/tasks-page-list-view.png) 

{{% /image_container %}}

このエラーを修正するには、正しいタイプのデータをページに渡すリストビュー内のボタンを配置することができます(リストビューの中にボタンを配置し、そのデータソースをエンティティ *SmartTask*に設定します)。 または、* タスク * ページのデータ ビューのデータ ソースをエンティティ **エンジニア** に変更します *エンジニア*。

| ------ | ----------------------------------------------------------------- | ---------------------------------------------------------- | ----------------------------------------------------------- |

## 6 画像ウィジェットの一貫性エラー

[静的イメージ ウィジェット](page-editor-widgets-images) の最も一般的な一貫性エラーは、イメージ ウィジェットをページに配置するときに発生する可能性があります。 実際の画像を選択しないでください 

[動的画像](page-editor-widgets-images) はデータコンテナ(データビューまたはリストビュー)内に配置する必要があり、エンティティを選択する必要があります。 

{{% image_container width="350" %}}![動的な画像のプロパティ](attachments/consistency-errors-pages/dynamic-image-properties.png) 

{{% /image_container %}}

静的画像と動的画像のエラーについては、以下の表を参照してください。 

| エラーコード | チェックパネルのテキスト                                 | エラーの原因                                                                | 修理方法                                                                                                                                    |
| ------ | -------------------------------------------- | --------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| CE0436 | 画像が選択されていません。                                | 静的イメージ ウィジェットをページに追加しましたが、イメージ自体が選択されていません。                           | 画像のプロパティ > **一般** セクションを開き、 **画像の選択** をクリックします。 デフォルトの画像を選択するか、独自の画像をアップロードできます。                                                        |
|        | データ ビューまたはリスト ビューなど、データ コンテナにこのウィジェットを移動します。 | 動的な画像をページに追加していることがありますが、データビューまたはリストビュー内にはありません。                     | 画像プロパティ > **一般** セクションを開き、 **新しいデータビューでラップ**をクリックします。 画像は自動的に新しいデータビュー内に配置されます。 リストビューまたはデータビューをページに追加し、その中に動的なイメージをドラッグ&ドロップすることもできます。 |
| CE0489 | このイメージ ビューアのデータ ソースの図形を選択します。                | ページに動的イメージを追加し、動的イメージはデータ ビューまたはリスト ビュー内に配置されます。 画像のエンティティが指定されていません。 | イメージ プロパティ > **一般** セクションを開き、 **エンティティ** フィールドでエンティティを選択します。                                                                            |




## 7クリックでアクション一貫性エラーが発生しました

ボタンや画像など、さまざまなウィジェットに **クリックアクション** を指定できます。 クリック操作の詳細については、 [ウィジェットのイベント](page-editor-widgets-events-section) セクションを参照してください。

クリック操作を完全に設定しない場合、最も一般的な一貫性エラーが表示されます。 たとえば、マイクロフローをクリック操作として選択しますが、マイクロフロー自体は選択しません。 

![クリックアクションエラー](attachments/consistency-errors-pages/on-click-action-error.png)

整合性エラーを修正するには、クリックアクションの設定を完了します (例えば、クリックアクション *ページ*の場合)。 開く特定のページを選択するか、クリックアクションを別のページに変更します。
