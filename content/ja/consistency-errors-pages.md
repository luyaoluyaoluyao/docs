## 1つの紹介

このドキュメントでは、Desktop Modelerでページを構成するときに発生する最も一般的な一貫性エラーを解決する方法を説明します。 整合性エラーの例として、ページ上のデータビューのentityプロパティを指定しない場合があります。

## 2つのリストビュー一貫性エラー

[リストビュー](data-sources) に [データソース](list-view) を適切に設定しないと、一貫性エラーが発生します。

以下のスキームは、リスト ビューのデータ ソースが **データベース**に設定されていることを示しています。 しかし、データベースから取得する必要がある特定のエンティティは設定されていません。 これにより一貫性エラーが発生します。

![データ ソースの整合性エラー・スキーム](attachments/consistency-errors-pages/dm-list-view-error.png)

以下の表は、リストビューの設定時に発生する最も一般的なエラーについて説明しています。 これらのエラーの原因とそれらの修正方法。

| チェックパネルのテキスト                                                         | エラーの原因                                                                            | 修理方法                                                                                   |
| -------------------------------------------------------------------- | --------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| このリスト ビューのデータ ソースに対して設定されたエンティティがありません。 エンティティを選択するか、データソースを変更します。   | リスト ビューのデータ ソースとして **データベース/XPath/Association** オプションが選択されていますが、エンティティが指定されていません。 | エラーを修正するには、次のいずれかを実行します。 <ul><li>リストビューを右クリックし、ドロップダウンメニューから**エンティティを選択**をクリックし、エンティティを設定します</li><li>リストビューのプロパティ > **データソース** を開き、**エンティティ(パス)** フィールド内のエンティティを選択します</li></ul>また、リスト ビューのデータ ソースのタイプを変更することもできます。   |
| このリスト ビューのデータ ソースにマイクロフローが設定されていません。 マイクロフローを選択するか、データソースを変更します。     | データ ソースが **Microflow**に設定されていますが、特定のマイクロフローは選択されていません。                            | リストビューのプロパティ > **データソース** を開き、 **Microflow** フィールドでマイクロフローを選択します。 または、データソースの種類を変更します。 |
| このリスト ビューのデータ ソースに設定されている nanoflow はありません。 ナノフローを選択するか、データソースを変更します。 | データソースは **Nanoflow**に設定されていますが、特定のナノフローは選択されていません。                                | リストビューのプロパティ > **データソース** を開き、 **Nanoflow** フィールドでマイクロフローを選択します または、データソースの種類を変更します。   |
| 属性 {AttributeName} はエンティティ {EntityName} の属性ではありません。                  | 内容を更新せずにリストビューのターゲットエンティティを変更しました。 リスト ビューの内容は、別のエンティティの属性で満たされます。                | ウィジェットのプロパティ > **データソース** を開き、 **属性 (パス)** の別の属性を選択します。                                |

## 3つのデータビューの一貫性エラー

適切な方法で [データ ビュー](data-sources) に [データ ソース](data-view) を構成しないと、一貫性エラーが発生します。 For example, you have selected **Listen to widget** as a data source, but you have not selected the specific **List widget** you are pointing to.

![リストウィジェットが設定されたデータビュー](attachments/consistency-errors-pages/dm-data-view-no-list-widget.png)

以下の表は、データビューの構成時に発生する最も一般的なエラーについて説明しています。 これらのエラーの原因とそれらの修正方法。

| チェックパネルのテキスト                                                          | エラーの原因                                                                                 | 修理方法                                                                                          |
| --------------------------------------------------------------------- | -------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| このデータ ビューのデータ ソースに設定されているエンティティがありません。 エンティティを選択するか、データソースを変更します。     | **コンテキスト** がデータ ビューのデータ ソースとして選択されていますが、エンティティが指定されていません。                              | データビューのプロパティ > **データ ソース** セクションを開き、 **エンティティ** フィールドでエンティティを選択します。 または、データソースの種類を変更します。      |
| このデータ ビューのデータ ソースに設定されているマイクロフローはありません。 マイクロフローを選択するか、データソースを変更します    | データソースとしてマイクロフローが選択されていますが、特定のマイクロフローは選択されていません。                                       | データ ビューのプロパティ > **データ ソース** セクションを開き、 **Microflow** フィールドでマイクロフローを選択します。 または、データソースの種類を変更します。 |
| このデータ ビューのデータ ソースに設定されているリスト ウィジェットがありません。 ウィジェットを選択するか、データソースを変更します。 | **ウィジェット** へのListen は、データ ビューのデータ ソースとして設定されています。 ただし、特定のリスト ビュー ウィジェットは同じページには存在しません。 | 同じページにリストビューを作成し、それを設定し、データ ビューのリスト ウィジェットとして選択します。 または、データソースの種類を変更します。                      |
| ページパラメータからオブジェクトを受信するすべてのデータビューは、同じエンティティを持つ必要があります。                  | データソースとして異なるエンティティを持つ複数のデータビューが1つのページにあります。                                            | すべてのデータビューで同じエンティティを選択するか、データソースを変更します。                                                       |

## 4 コンテキストが利用できない一貫性エラー

利用できないコンテキストがページに期待されている場合に発生する可能性のあるエラーについては、下の表に記載されています。

| チェックパネルのテキスト                                                           | エラーの原因                                                                                                                                                                             | 修理方法                                                                                                                |
| ---------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| 選択されたページ {Name of the page} は、 {type of object}型のオブジェクトを想定しています。       | このページには、特定の型のオブジェクトが渡されることを期待するデータビューがあります。 このエラーは、このオブジェクトが使用可能な別のページからページが呼び出されたときに発生します。 より詳細な例については、 [4.1 Error Fix Example 1](#error-example-1) を参照してください。                    | オブジェクトが設定されたデータビューを持つページに渡されていることを確認します。 詳しい情報については、  [4.1 Error Fix Example 1](#error-example-1) を参照してください。        |
| 選択されたページ {Name of page} は、ここで利用可能な Y 型のオブジェクトと互換性のない型 X のオブジェクトを期待します。 | ページを開くウィジェット(ボタンなど)があること。 このページには、特定の型のオブジェクトが渡されることを期待するデータビューがあります。 ただし、ウィジェットは別のタイプのオブジェクトを持つデータコンテナ内に配置されます。 詳細な例については、 [4.2 Error Fix Example 2](#error-example-2) を参照してください。 | ボタンが正しいデータコンテナに配置されていることを確認し、正しいオブジェクトの種類をページに渡します。 詳細については、 [4.2 Error Fix Example 2](#error-example-2) を参照してください。 |

### 4.1 エラー修正例 1 {#error-example-1}

呼び出し元のページやマイクロフローから渡されないコンテキストをページが期待すると、一貫性エラーが発生します。

Let us study an example: the **Customers** page contains a list view with a list of all customer names (**Customer** is set as **Entity** in the **Data Source** properties), and a **Details** button outside of the list view (placed in a [container](container) only). **詳細** ボタンは、ユーザーがクリックすると **顧客詳細** ページを開きます (ボタンの **クリック時** は **ページ** に設定されています)。

![顧客ページ上のボタンのプロパティ](attachments/consistency-errors-pages/dm-customers-page.png)

ただし、 **Customer Details** ページには、オブジェクト *Customer* が渡されることを期待するデータ ビューがあります。 言い換えれば、このページはデータを最初に表示できるようにする必要があります。

![データビューが顧客オブジェクトを期待する](attachments/consistency-errors-pages/dm-data-view-customer.png)

このオブジェクトは **顧客** ページから渡されていないため、一貫性エラーが発生します。

![コンテキストが利用できない場合のエラーの例](attachments/consistency-errors-pages/dm-object-error.png)

**顧客** ページの **詳細** ボタンはデータコンテナの外側にあります。 どのお客様(オブジェクト)に合格すべきか分かりません。 このエラーを修正する方法は、以下によって異なります:

* You want to pass a specific *Customer* object from the Customer list to the **Customer Details** page, in other words, the details of a particular customer will be displayed on the **Customer Details** page (for more information, see section [4.1.1 Passing a Specific Object to the Page](#passing-specific-object))
* You want to create a new object of the *Customer* type and pass it to the **Customer Details** page, this means that a new customer will be created (for more information, see section [4.1.2 Creating a New Object and Passing it to the Page](#creating-new-object))

#### 4.1.1 特定のオブジェクトをページに渡す {#passing-specific-object}

特定の顧客の詳細を開くために **顧客詳細** ページをご希望の場合 つまり、特定のオブジェクトをページに渡したいということです。 すでに **顧客** ページに顧客リストが表示されているため、次のようにこのエラーを修正できます。

1. **顧客** ページを開きます。

2.  リスト ビュー内の **詳細** ボタンをドラッグします。

    ![詳細ボタンの例](attachments/consistency-errors-pages/dm-details-button-inside-the-list-view.png)

Now the button gets the object of type *Customer* from the list view on the **Customers** page, and it will be passed to the **Customer Details** page. その結果、特定の顧客の詳細が **顧客詳細** ページに表示されます。

#### 4.1.2 新しいオブジェクトを作成し、ページに渡す。 {#creating-new-object}

新規顧客を作成し、 **顧客詳細** ページに顧客の詳細を記入する場合。 次のことができます。

1. **顧客** ページを開きます。

2. **Details** ボタンのプロパティを開き、 **Create Object** を **On Click Action** として設定します。

3. **顧客** を **エンティティ** として設定します。

4.  **顧客詳細** を **ページ** として設定します。

    ![クリックイベントの例で](attachments/consistency-errors-pages/dm-button-create-object.png)

5. ボタンのキャプションを **詳細** から **追加**に変更する このボタンは、既存の顧客の詳細を表示する代わりに新しい顧客を作成するようになります。

ユーザーがこのボタンをクリックすると、 **Customer Details** ページが開き、新しい *Customer* オブジェクトが作成されます。

### 4.2 エラー修正例 2 {#error-example-2}

ウィジェットがページを開き、このウィジェットがエンティティ X のデータ コンテナ内にある場合。 しかし、参照ページはエンティティYを期待しており、一貫性エラーが発生します。

例えば、 **タスク** ページを開く **エンジニア** ページに **詳細** ボタンがあります。

![エンジニアページのボタン](attachments/consistency-errors-pages/dm-engineers-page.png)

The button is placed inside a list view; the list view's data source is set to entity *Engineer* in **Properties** > **Data Source**.

The Tasks page has a data view on it, but the data view's data source is set to entity *SmartTask* in **Properties** > **Data Source**.

This means that data view expects the object of type *SmartTask* passed to it, but the **Engineers** page is passing the object of type *Engineer*.

![データソースの例](attachments/consistency-errors-pages/dm-tasks-page-list-view.png)

このエラーを修正するには、次のいずれかを実行できます。

* Place the **Details** button within a data container that will pass the correct type of data to the page (for example, you can place the button inside the list view and set its data source to entity *SmartTask*)
* 任意のオブジェクトが渡されることを期待しない別のページを選択するか、 *Engineer* 型のオブジェクトを期待します。
* **タスク** ページのデータ ビューのデータ ソースをエンティティ *エンジニア* に変更します

### 5.1 参照セレクターの多重度が正しくありません {#incorrect-multiplicity-reference}

参照セレクタは、1対多の関連付けを表示および編集するために使用されるウィジェットです。 ウィジェットの詳細については、 [リファレンスセレクター](reference-selector) を参照してください。

例えば、あなたは彼らが働く1つの都市に関連付けられている複数の従業員を持っている。 これは1対多の関連付けです。複数の *Employees* オブジェクトは1つの *都市* オブジェクトに関連付けられています。  Associations that refer to a single object in this manner are *references*, as opposed to *reference sets*, in which multiple objects can refer to multiple other objects. 参照では、関連付けの "単一オブジェクト" 側は常に関連付けの所有者です。 関連付けとそのタイプの詳細については、 [Associations](associations) を参照してください。

![1対多の協会](attachments/consistency-errors-pages/dm-many-to-one-association.png)

間違ったタイプの関連を持っている場合 一貫性エラーが表示されます: *アソシエーション {Name} は参照(参照セットではない)でなければなりません*。

このエラーを修正するには、次のいずれかを実行します:

1. ドメインモデルを開き、参照セレクタで使用している関連付けをダブルクリックします。

2.  **関連付けのプロパティ** ダイアログウィンドウで、 **Multiplicity** を 1 対 多数に変更します (この例では、次のようにします)。 複数の 'Employee' オブジェクトは 1 つの 'City' オブジェクトに関連付けられています)。

    ![1対多の関連付けの多重性](attachments/consistency-errors-pages/dm-one-to-many-multiplicity.png)

3. **OK** をクリックして変更を保存します。

関連の多重度を変更し、エラーを修正しました。

### 5.2 リファレンスセットセレクタと入力リファレンスセットセレクタの多重度が不正です {#incorrect-multiplicity-reference-set}

リファレンスセットセレクタと入力セットセレクタは、多対多の関連付けを表示および編集するために使用されるウィジェットです。 これらのウィジェットの詳細については、 [Reference Set Selector](reference-set-selector) および [Input Reference Set Selector](input-reference-set-selector) を参照してください。

たとえば、週には複数の従業員が異なる都市の顧客を訪問することができます。 従って、多くの従業員は多くの都市に関連付けられています これは *従業員* エンティティと *都市* エンティティの間の多対多の関連です（複数の従業員は複数の都市に関連付けられています）。 この方法で複数のオブジェクトを参照する関連付けは、 *参照集合* です。 関連付けとそのタイプの詳細については、 [Associations](associations) を参照してください。

![多対多の協会](attachments/consistency-errors-pages/dm-many-to-many-association.png)

間違ったタイプの関連を持っている場合 一貫性エラーが表示されます: *関連付け {Name} は参照セットでなければなりません*.

エラーを修正するには、次のいずれかを実行します。

1. ドメインモデルを開き、参照セットセレクタまたは入力参照セットセレクタに使用している関連付けをダブルクリックして、以下を行います: <br/>

2.  **関連付けのプロパティ** ダイアログウィンドウで、 **多重度** を多対多に変更します (この例では、 複数の 'Employee' オブジェクトは複数の 'City' オブジェクトに関連付けられています)。

    ![多対多の関連付けの多重性](attachments/consistency-errors-pages/dm-changing-multiplicity.png)

3. **OK** をクリックして変更を保存します。

関連付けの多重度を変更し、エラーを修正しました。

### 5.3 リファレンスセットセレクタと入力リファレンスセットセレクタの所有権が正しくありません {#incorrect-ownership}

参照セットセレクタと入力参照セットセレクタは関連付けの表示と編集に使用されます。 所有権はそれを構成する上で重要な財産です オーナーシップの詳細については、 [Associations](associations) を参照してください。

関連付けの所有者を変更すると、一貫性エラーが表示されます: * {Name} には関連 {Name}がありません*。 所有者は、参照集合セレクタまたは入力参照集合セレクタが指すものでなければなりません。

ドメインモデルに2つのエンティティがある場合の例を見てみましょう: **都市** と **従業員**。

矢印は常にウィジェットに向かってコンテキストから離れています。 従業員に都市を選択させたい場合は、関連付けの所有者は *従業員* エンティティである必要があります。 参照セットセレクタまたは入力参照セットセレクタを配置するデータコンテナ(データビューなど)に注意してください。 データソースを **コンテキスト** >**従業員** に設定する必要があります。

![データビューのプロパティ: 従業員エンティティ](attachments/consistency-errors-pages/dm-data-view-employee-context.png)

各都市に所属する従業員を表示したい場合は、 *都市* エンティティを所有者にする必要があります。 参照集合セレクタまたは入力参照集合セレクタを配置するデータコンテナ (データビューなど) データソースを **コンテキスト** >**シティ** に設定する必要があります。

![データビューのプロパティ: 都市エンティティ](attachments/consistency-errors-pages/dm-data-view-city-context.png)

両方を組み合わせる場合は、所有権は *両方の* に設定する必要があります。 次に、参照セットセレクタまたは入力参照セットセレクタをデータ コンテナに *City* または *Employee* エンティティに設定することができます。

所有権を変更するには、以下を行います:

1. ドメインモデルを開きます。

2. 参照セットセレクターまたは入力参照セットセレクターで関連付けを使用するエンティティのいずれかをダブルクリックして、プロパティを開きます。

3.  In the entity properties > the **Associations** tab, change the **Owner** to **Default** or **Both** depending on your use-case.

    ![所有者プロパティ](attachments/consistency-errors-pages/dm-selecting-the-owner.png)

関連付けの所有者を変更しました。

## 7 画像ウィジェットの整合性エラー

画像ウィジェットをページに追加しても、画像自体が選択されていない場合、整合性エラーが表示されます。

修正するには、イメージ プロパティ > the **General** section > the **Image** field and select an image. 画像ウィジェットの詳細については、 [イメージ](image) を参照してください。

## 8クリックでイベントの一貫性エラーが発生しました

ボタンや画像など、さまざまなウィジェットに **イベント** を指定できます。

最も一般的な整合性エラーは、click イベントを完全に構成していないと接続されます。

整合性エラーを修正するには、クリックイベントの構成を完了します (例: for an on click event **Show a page**, select a specific page that should open), or change the on click actions
