## 1つの紹介

このドキュメントでは、Mendix Studio でページを構成する最も一般的な一貫性エラーを解決する方法を説明します。 ページの詳細については、 [ページ](page-editor) を参照してください。

整合性エラーの例として、ページ上のデータビューのentityプロパティを指定しない場合があります。

{{% alert type="info" %}}

このドキュメントでは、 *すべてのエラー* を記述していません。多くのエラーが発生する可能性があります。 そのうちのいくつかは簡単で、余分な説明は必要ありません、他のものはまれおよび/または大きくユースケースに依存しています。

{{% /alert %}}

一部のエラーにはエラーコードがあり、これらのエラーがドキュメントに記載されている場合、Studio には対応するドキュメントへのクリック可能なリンクがあります。 この場合、エラーコードがない人もいます。 特定のエラーがドキュメントに記述されているかどうかを手動で検索できます( **チェック** パネルに表示されるメッセージで検索できます)。

以下の表は、リストビューの設定時に発生する最も一般的なエラーについて説明しています。 間違いの原因や解決方法などです

| CE0488 | このリスト ビューのデータ ソースに対して設定されたエンティティがありません。 エンティティを選択するか、データソースを変更します。   | The **Database** (or **XPath**/**Association**) option is selected as the data source for a list view on the page has been set, but the **Entity** property is not specified. | 次のいずれかを実行します。 <ul><li>リストビューのプロパティ > **データソース** を開き、**エンティティ** フィールド内のエンティティを選択します</li><li>データソースのタイプを変更する</li></ul> |

### 4.1 CE1568 のエラー修正例 {#error-example-1}

ページが別のページやマイクロフローから渡されないコンテキストを期待すると、一貫性エラーが発生します。

For example, the **Customers** page contains a list view with a list of all customer names (**Customer** is set as **Entity** in the **Data Source** properties), and a **Details** button outside of the list view (placed in a [container](page-editor-widgets-structure#container-overview) only). **詳細** ボタンは、ユーザーがクリックすると **顧客詳細** ページを開きます ( **クリック時** ボタンが **ページ** に設定されており、 **オブジェクトの作成** オプションが無効になっています)。

![顧客ページ上のボタンのプロパティ](attachments/consistency-errors-pages/customers-page.png)

ただし、 **Customer Details** ページには、オブジェクト *Customer* が渡されることを期待するデータ ビューがあります。 言い換えれば、このページはデータを最初に表示できるようにする必要があります。

{{% image_container width="350" %}}![データビューが顧客オブジェクトを期待する](attachments/consistency-errors-pages/data-view-customer.png)
{{% /image_container %}}

このオブジェクトは **顧客** ページから渡されていないため、一貫性エラーが発生します。

![コンテキストが利用できない場合のエラーの例](attachments/consistency-errors-pages/object-error.png)

**顧客** ページの **詳細** ボタンは、データコンテナ（データビューまたはリストビュー）の外側にあるため、どのオブジェクトを渡すべきかはわかりません。 このエラーを修正する方法は、以下によって異なります:

* You want to pass a specific *Customer* object to the **Customer Details** page, in other words, the details of a particular customer will be displayed on the **Customer Details** page (for more information, see the [Passing a Specific Object to the Page](#passing-specific-object) section)
* You want to create a new object of the *Customer* type and pass it to the **Customer Details** page, this means that a new customer will be created (for more information, see the [Creating a New Object and Passing it to the Page](#creating-new-object) section)

#### 4.1.1 特定のオブジェクトをページに渡す {#passing-specific-object}

特定の顧客の詳細を開くために **顧客詳細** ページをご希望の場合 つまり、特定のオブジェクトをページに渡したいということです。 すでに **顧客** ページに顧客リストが表示されているリスト表示があるため、次の方法でこのエラーを修正できます。

1. **顧客** ページを開きます。

2.  リスト ビュー内の **詳細** ボタンをドラッグします。

    {{% image_container width="350" %}}![リストビューの例](attachments/consistency-errors-pages/list-view-content.png)
    {{% /image_container %}}


Now the button gets the object of type *Customer* from the list view on the **Customers** page, and it will be passed to the **Customer Details** page. その結果、特定の顧客の詳細が **顧客詳細** ページに表示されます。

#### 4.1.2 新しいオブジェクトを作成し、ページに渡す。 {#creating-new-object}

新規顧客を作成し、 **顧客詳細** ページに顧客の詳細を記入する場合。 次のことができます。

1. **顧客** ページを開きます。

2. **詳細** ボタン > **イベント** セクションのプロパティを開き、 **オブジェクトの作成** オプションを有効にします。

     {{% image_container width="350" %}}![オブジェクトの作成オプションが有効になっています](attachments/consistency-errors-pages/create-object-option-enabled.png)
     {{% /image_container %}}

3. **顧客** を **エンティティ** として設定します。

4. ボタンのキャプションを **詳細** から **追加**に変更する このボタンは新規顧客を作成し、既存の顧客の詳細を表示しません。

ユーザーがこのボタンをクリックすると、 **Customer Details** ページが開き、新しい *Customer* オブジェクトが作成されます。

### 4.2 CE1569 のエラー修正例 {#error-example-2}

ウィジェットがページを開き、このウィジェットがエンティティ X のデータ コンテナ内にある場合。 しかし、参照ページはエンティティYを期待しており、一貫性エラーが発生します。

たとえば、 **タスク** ページを開く **Engineers** ページに **New** ボタンがあります。

{{% image_container width="350" %}}![エンジニアページのボタン](attachments/consistency-errors-pages/engineers-page.png)
{{% /image_container %}}

The button is placed inside a list view; the list view's data source is set to entity *Engineer* in **Properties** > **Data Source**.

The Tasks page has a data view on it, but the data view's data source is set to entity *SmartTask* in **Properties** > **Data Source**.

This means that data view expects the object of type *SmartTask* passed to it, but the **Engineers** page is passing the object of type *Engineer*. 結果として一貫性エラーが発生します。

{{% image_container width="350" %}}![タスク ページの一覧表示](attachments/consistency-errors-pages/tasks-page-list-view.png)
{{% /image_container %}}

このエラーを修正するには、次のいずれかを実行します:

* 正しい種類のデータをページに渡すリストビューにボタンを配置します(リストビューの中にボタンを配置し、そのデータソースをエンティティ *SmartTask*に設定します)
* オブジェクトが渡されるとは思わないボタンの別のページを選択します または、 *Engineer* 型のオブジェクトを期待します
* **タスク** ページのデータ ビューのデータ ソースをエンティティ *エンジニア* に変更します

## 5つのデータ整合性エラー

データソースとしてオブジェクトを期待するウィジェットがそれを取得したり、別のエンティティタイプからオブジェクトを取得したりしない場合。 データの一貫性が間違ってしまいます

このタイプの最も一般的なエラーのいくつかは以下の表に記載されています:

| エラーコード | チェックパネルのメッセージ                                                                                                                                | エラーの原因                                                                                                           | 修理方法                                                                                                                                                     |
| ------ | -------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| CE0552 | Microflow {name of the microflow} はオブジェクトを返しません。                                                                                             | ウィジェットのデータ ソース (データビューなど) が **Microflow**に設定されています しかしマイクロフローは物体を返しません                                           | マイクロフローを開き、戻り値を設定します。 次の操作を行います: <ol><li>終了イベントを選択します。</li><li>プロパティでは、**Nothing** から **Value** を**Value** に設定します。</li><li>**Data Type**を設定し、必要に応じて**Entity**オプションを設定します（このオプションが表示されるかどうかは選択したデータ型によって異なります）。</li><li> **値**を設定します。</li></ol>                                                                                                |
| CE0551 | Microflow {name of the microflow} はリストを返しません。                                                                                                | リスト ビューのデータ ソースは **Microflow**に設定されていますが、マイクロフローはリストを返しません。                                                      | マイクロフローを開き、リストを返すように構成します。 次の操作を行います: <ol><li>終了イベントを選択します。</li><li> プロパティでは、**Nothing** から **Value** を**Value** に設定します。</li><li>**Data Type** を *List* に設定します。</li><li>**エンティティ** オプションを設定します。</li><li>**値**を設定します。</li></ol>                                                                                           |
| CE1573 | 選択したマイクロフローにはパラメータ {Name of the parameter} がありますが、ページ上に利用可能なデータはありません。 {Name of the widget} をデータソースとして {Name of the entity} をデータコンテナ内に配置します。 | ウィジェットのオンクリックアクション (ボタンなど) としてマイクロフローを選択し、マイクロフローにパラメーターが含まれていること。 しかし、ウィジェットがマイクロフローに渡すデータ(例えば、オブジェクト)は利用できません。 | データコンテナにウィジェットを配置し、データコンテナのデータソースが microflow パラメータの **Entity** プロパティと一致することを確認します。 詳細な例と修正については、 [CE1573のエラー修正例](#error-fix-example-3) セクションを参照してください。   |
| CE1574 | 選択されたマイクロフローにはパラメータ {Name of the parameter} がありますが、使用可能なページ ( {name of the widget}を囲む) にはデータがありません。 利用可能なデータは {list of data available} です。   | ウィジェットのデータ ソースとしてマイクロフローを選択しましたが、データ (例: このウィジェットで使用可能なオブジェクト)は、マイクロフローのパラメータと一致しません。                            | ウィジェットで使用可能なオブジェクトが、microflow パラメータの **Entity** プロパティと一致していることを確認します。 詳細な例と修正については、 [Error Fix Example for CE1574](#error-fix-example-4) セクションを参照してください。 |

### 5.1 CE1573のエラー修正例 {#error-fix-example-3}

マイクロフローをウィジェットのオンクリックアクションとして設定する場合 このマイクロフローは、利用できないオブジェクトを期待し、エラーになります。

例えば、 *顧客* という名前のページには、マイクロフローを呼び出すボタンがあります (ボタンの **クリックアクション** は *Microflow* に設定されています):

 ![](attachments/consistency-errors-pages/button-on-click-action.png)

ただし、マイクロフローにはパラメータ *顧客* が含まれています:

![](attachments/consistency-errors-pages/microflow-parameter.png)

microflow パラメータはオブジェクト *顧客*を期待します ボタンのあるページではこのオブジェクトは使用できないのでエラーになります

これを修正するには、次の手順を実行します。

1.  *顧客* ページを開き、データコンテナをドラッグ&ドロップします。 たとえば、リストビューをドラッグ&ドロップできます。

2.  リスト ビューのデータ ソースを *データベース* に設定し、 **エンティティ** を *顧客* に設定します。

    ![](attachments/consistency-errors-pages/list-view-data-source.png)

3. ボタンをリストビュー内に配置します。

*Customer* オブジェクトがページに表示され、microflow パラメータ *Customer* と一致します。

### 5.2 エラー修正例 CE1574 {#error-fix-example-4}

マイクロフローをウィジェットのオンクリックアクションとして設定すると、このマイクロフローは特定のデータを期待します (例えば、マイクロフローは特定のデータを期待します)。 異なるオブジェクトがウィジェットに使用できるため、エラーが発生します。

For example, on a page called *Customers* you have a button that calls a microflow (meaning, an **On Click Action** of the button is set to *Microflow*):

![](attachments/consistency-errors-pages/button-on-click-action.png)

マイクロフローには、パラメータ *顧客* が含まれます :

![](attachments/consistency-errors-pages/microflow-parameter.png)

*顧客* ページには、たとえば、オブジェクト *写真* を持つデータビューなどのデータコンテナもあります。 つまり、データ ビューのデータ ソースが *コンテキスト* に設定されており、 **エンティティ** が *写真* に設定されています。

![](attachments/consistency-errors-pages/data-view-photo-data-source.png)

As the microflow has the parameter *Customer*, and the data view has the object *Photo*, they are conflicting and resulting into an error.

このエラーを修正する最善の方法は、マイクロフローを変更して *Photo* を受け入れるか、別のエンティティのボタンをデータコンテナに入れることです。

## 6 入力要素の整合性エラー

[入力要素](page-editor-widgets-input-elements) の最も一般的なエラー (テキストボックス、ドロップダウンなど) チェックボックス等の原因や修正方法については下表に記載しています。

| エラーコード | チェックパネルのメッセージ                                                     | エラーの原因                                                     | 修理方法                                                              |
| ------ | ----------------------------------------------------------------- | ---------------------------------------------------------- | ----------------------------------------------------------------- |
| CE0544 | このウィジェットはデータコンテナ内でのみ機能できます。 データ ビュー、リスト ビュー、またはテンプレート グリッドに移動します。 | 入力ウィジェットをページに追加していることがありますが、それはデータ ビューまたはリスト ビュー内にはありません。  | エラーを修正する方法は2つあります: <ul><li>特定のデータビューまたはリストビューがまだ利用できない場合: ウィジェットのプロパティ > **データソース** を開き、**新しいデータビュー**でラップをクリックします。 入力ウィジェットは自動的に新しいデータビュー内に配置されます。</li><li>データビューまたはこの入力要素に関連するリストビューがすでに存在する場合: データビューまたはリストビューの中に入力要素をドラッグアンドドロップ</li></ul>                       |
| CE0545 | この {name of the input element} の属性を選択します。                         | 入力要素を追加し、それがデータコンテナ内にあります。 ただし、この入力要素が接続されている属性は選択されていません。 | ウィジェットのプロパティ > **Data Source** を開き、 **Attribute** フィールドで属性を選択します。 |

## 7 画像ウィジェットの整合性エラー

[静的イメージ ウィジェット](page-editor-widgets-images-and-files) の最も一般的な一貫性エラーは、イメージ ウィジェットをページに配置するときに発生する可能性があります。 実際の画像を選択しないでください

[動的画像](page-editor-widgets-images-and-files) はデータコンテナ(データビューまたはリストビュー)内に配置する必要があり、エンティティを選択する必要があります。

{{% image_container width="350" %}}![動的な画像のプロパティ](attachments/consistency-errors-pages/dynamic-image-properties.png)
{{% /image_container %}}

静的画像と動的画像のエラーについては、以下の表を参照してください。

| エラーコード | チェックパネルのメッセージ                                | エラーの原因                                                                | 修理方法                                                                                                                                    |
| ------ | -------------------------------------------- | --------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| CE0436 | 画像が選択されていません。                                | 静的イメージ ウィジェットをページに追加しましたが、イメージ自体が選択されていません。                           | 画像のプロパティ > **一般** セクションを開き、 **画像の選択** をクリックします。 デフォルトの画像を選択するか、独自の画像をアップロードできます。                                                        |
|        | データ ビューまたはリスト ビューなど、データ コンテナにこのウィジェットを移動します。 | 動的な画像をページに追加していることがありますが、データビューまたはリストビュー内にはありません。                     | 画像プロパティ > **一般** セクションを開き、 **新しいデータビューでラップ**をクリックします。 画像は自動的に新しいデータビュー内に配置されます。 リストビューまたはデータビューをページに追加し、その中に動的なイメージをドラッグ&ドロップすることもできます。 |
| CE0489 | このイメージ ビューアのデータ ソースの図形を選択します。                | ページに動的イメージを追加し、動的イメージはデータ ビューまたはリスト ビュー内に配置されます。 画像のエンティティが指定されていません。 | イメージ プロパティ > **一般** セクションを開き、 **エンティティ** フィールドでエンティティを選択します。                                                                            |

## 8クリックでアクションの一貫性エラーが発生しました

ボタンや画像など、さまざまなウィジェットに **クリックアクション** を指定できます。 クリックアクションの詳細については、 [イベント セクション](page-editor-widgets-events-section) を参照してください。

クリック操作を完全に設定しない場合、最も一般的な一貫性エラーが表示されます。 たとえば、マイクロフローをクリック操作として選択しますが、マイクロフロー自体は選択しません。

![クリックアクションエラー](attachments/consistency-errors-pages/on-click-action-error.png)

整合性エラーを修正するには、クリックアクションの設定を完了します (例えば、クリックアクション *ページ*の場合)。 開く特定のページを選択するか、クリックアクションを別のページに変更します。
