## 1つの紹介

ワークフローは、Mendix StudioとMendix Studio Proの視覚的な言語であり、プロセスに関係するビジネス上の問題を解決することができます。 これは、マイクロフローエディタやページエディタなどの他の視覚言語と完全に統合されています。

ワークフローと [microflow](microflows) の主な違いは、待機の側面です – ワークフローはエンドユーザーからの入力が得られるまで一時停止されます。 例えば、 従業員がトラベルリクエストを送信し(ワークフローの開始をトリガー)、ボタンをクリックしてマネージャがリクエストを承認するまでワークフローは一時停止されます。

Studio でアプリのワークフローを表示するには、左側のメニューバーの **ワークフロー** アイコンをクリックします。

![ワークフローアイコン](attachments/workflows/workflow-icon.jpg)

ワークフローは、アプリケーションのロジックを視覚的に処理する方法です。 ワークフローはフローチャートのようになります。 新しいワークフローでは、 *開始イベント* (ワークフローの開始点) と *終了イベント* (ワークフローの終了イベント) がデフォルトで作成されます。 *path* と呼ばれるワークフローのフローにさまざまなアクティビティを追加できます。

![ワークフローの例](attachments/workflows/workflow-example.jpg)

## 2 ワークフローアプリテンプレート

ワークフローを使用するための出発点として、ワークフロー固有のアプリテンプレートを使用できます。 たとえば、アプリケーションが作成されるエンドユーザーに対して、承認リクエストフォームを構成できます。 ダッシュボード、管理者ページ、ダッシュボード、ワークフローなど、事前に構成された要素が含まれており、後でさらにカスタマイズできます。 新しいアプリを作成すると、これらのテンプレートが表示されます。

## 3 基本機能の実行

ワークフローを操作する際に、以下の基本的な機能を実行できます。

* [ワークフローを開く](#open)
* [ワークフローを作成](#create)
* [ワークフローを複製](#duplicate)
* [ワークフローをコピーして貼り付け](#copy-paste)
* [ワークフローを削除](#delete)
* [ワークフローに要素を追加](#add-elements)

### 3.1 ワークフローを開く {#open}

Studio でワークフローを開くには、次の手順を実行します。

1. 左のメニューバーにあるワークフローアイコンをクリックします。

2. 表示されるワークフローのリストで、開きたいワークフローを選択してクリックします。


選択したワークフローが開きます。

### 3.2 ワークフローの追加 {#create}

Studio でアプリにワークフローを追加するには、次の手順を実行します。

1. 左のメニューバーにあるワークフローアイコンをクリックします。

2. 新しいワークフローを追加したいモジュールを選択し、このモジュールの横にあるプラスアイコンをクリックします。

    ![新しいワークフロー](attachments/workflows/new-workflow.jpg)

    モジュールの詳細については、 [Domain Model](domain-models) を参照してください。

3. In the **Create new workflow** dialog box, fill in the name of the workflow and select a workflow entity (for more information on type of entities, see the [Entities and Their Types](domain-models#entity-types) section in *Domain Model*):

    ![新しいワークフローを作成](attachments/workflows/create-new-workflow.jpg)

4. **Create** をクリックします。

ワークフローが作成されます。

### 3.3 ワークフローを複製 {#duplicate}

ワークフローを複製するには、次の操作を行います。

1. 左のメニューバーの **ワークフロー** アイコンをクリックします。

2. サイドパネルで省略記号アイコンをクリックし、ドロップダウンメニューから **Duplicate** を選択します。

    ![ワークフローを複製](attachments/workflows/duplicate.jpg)

ワークフローが重複しています。

### 3.4 ワークフローのコピーと貼り付け {#copy-paste}

ワークフローをコピー&ペーストするには、次の手順を実行します。

1. 左のメニューバーの **ワークフロー** アイコンをクリックします。

2. サイドパネルで省略記号アイコンをクリックし、ドロップダウンメニューから **Copy to clipboard** を選択します。

    ![ワークフローをコピー](attachments/workflows/copy.jpg)

3. ワークフローを貼り付けるスタジオアプリを開き、 <kbd>Ctrl</kbd> +<kbd>V</kbd> または <kbd>Cmd</kbd> +<kbd>V</kbd> を押します。

ワークフローが貼り付けられました。 Studio のコピー/貼り付け機能の詳細については、 [一般情報](general#copy-paste-documents) の *コピー/貼り付け ワークフロー、ページ、マイクロフロー、および 列挙数* セクションを参照してください。

### 3.5 ワークフローの削除 {#delete}

Studio でワークフローを削除するには、次のいずれかを実行します。

1. 削除したいワークフローを開き、以下の手順に従ってください。
    1. **プロパティ** タブを開きます。
    2. **プロパティ** タブの下部にある **削除** をクリックします。
2. 左側のメニューバーのワークフローアイコンをクリックし、次の操作を行います:
    1. サイドパネルで省略記号アイコンをクリックし、ドロップダウンメニューから **Delete** を選択します。

選択したワークフローが削除されます。

### 3.6 ワークフローへの要素の追加 {#add-elements}

ワークフローに要素を追加するには、次の操作を行います。

1. **Toolbox** タブを開きます。
2. ワークフローパスにこの要素を追加およびドラッグ&ドロップしたい要素を選択します。

選択した要素が追加されます。

## 4 つのツールボックス要素

**Toolbox** タブには、パス上にドラッグ&ドロップできる要素が含まれています。 以下は、すべての要素のカテゴリ概要です。 次のセクションが使用されます:

* [全般](#general)
* [ユーザーアクション](#user-actions)
* [システムアクション](#system)

### 4.1 全般 {#general}

**一般** セクションの要素は、ワークフローのパスを制御するのに役立ちます。たとえば、並列パスを追加するか、終了するなどです。

![一般セクション](attachments/workflows/general.jpg)

このセクションの要素は以下の表で説明されています:

| 要素                                                  | 説明                                                                                                                                                                                                                                                          |
| --------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| イベントを開始                                             | ワークフローの開始点。 ワークフローは、ページ上の [コールワークフロー](page-editor-widgets-events-section#call-workflow) をオンクリックするか、マイクロフローの [ワークフローコール](microflows#microflow-workflow-activities) アクションによってトリガーされます。 <br /> [ワークフロー プロパティ](workflow-properties) を開くには、開始イベントをクリックします。 |
| [決定](workflows-general-activities#decision)         | 条件に基づいて選択を行い、発信パスの1つだけに従います。                                                                                                                                                                                                                                |
| [ジャンプアクティビティ](workflows-general-activities#jump)    | ワークフロー内の他のアクティビティにジャンプできます。                                                                                                                                                                                                                                 |
| [並列分割](workflows-general-activities#parallel-split) | ワークフローに 2 つ以上の並列パスを追加します。                                                                                                                                                                                                                                   |
| [アクティビティを終了する](workflows-general-activities#end)    | ワークフローのパスを終了します                                                                                                                                                                                                                                             |

### 4.2 ユーザーアクション {#user-actions}

[ユーザー タスク](workflows-user-task) – 特定のユーザーまたはフィルターまたはマイクロフローを使用してユーザーのグループにタスクを割り当てることができるワークフローの中心的な要素。

![ユーザーアクション](attachments/workflows/user-actions.jpg)

### 4.3 システムアクション {#system}

[](workflow-system-actions) アクティビティを選択したマイクロフローを呼び出します。 このアクティビティを使用して、ユーザーの操作を必要としないワークフローのパスにアプリケーションロジックを追加できます。

![システムアクション](attachments/workflows/system-actions.jpg)
