本文档描述了什么页面和什么样的部件可以放置在它们上。 页面、 代码片段或布局文档的属性可以在那些文档类型的文档页面上找到： [页面](page) [代码片段](snippet)和 [布局](layout)

| 类型                                                                   | 描述                                                                                                                                           |
| -------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| [![](attachments/pages/page-icon.png)](页面) 页 次                       | [页面](page) 定义Mendix 应用程序的最终用户界面。 除了页外，在创建界面时还会出现另外两种类型的文件：布局和片段。 页面是指实际显示给最终用户的东西。 代码片段和布局是创建页面的构建块，但它们不会直接显示给用户。                            |
| [![](attachments/pages/layout-icon.png)](布局) 布局                      | [布局](layout) 指定了什么位置。 每个页面都是基于一个布局。 布局包含基于该布局的每个页面上都放置的部件和结构。 例如，通常在布局上放置菜单栏小部件，以便它在所有页面上可见。                                                 |
| [![](attachments/pages/snippet-icon.png)](snippet) 片段                | [代码片段](snippet) 定义可重复使用的接口部件。 它们可以用于页面和布局。 通过使用代码片段，如果你想要修改接口，你必须在更少的地方进行更改。 例如，您可以有一个代码片段用于模板网格的内容区域和数据视图。 如果你在代码片段中添加一行到表中，这种更改将会在这两个地方出现。 |
| [![](attachments/pages/building-block-icon.png)](building-block) 建筑块 | [Building block](building-block) 允许用户轻松地点击带有预定样式小部件的页面。 构建模块将自动出现在页面编辑器工具框中，可以轻松重新使用。 通过设计一个综合的构件库，可以避免大量设计页数所涉及的迷信工作。                       |
| [![](attachments/pages/page-template-icon.png)](page-templates) 页面模板 | 新创建的页面基于 [页面模板](page-templates)。 绘制共同设计图案， 创建一个新页面所涉及的大量初始工作可以通过设置一套适当的页面模板来简化。                                                              |

页面、布局和代码片段使用小部件构建。 有许多类型的小部件，并非每个小部件都可以用于所有的三种文档类型。 布局支持为页面提供结构的小部件，但不是显示数据的小部件。 这样做是为了明确安排的意图：它们应该界定什么是什么，而不是更多的东西。 然而，代码片段可以放置在布局中，这是将更多类型的部件放入布局的间接方式。

部件分为以下类别：

* [菜单小部件](menu-widgets) 允许用户浏览应用程序
    * 示例：菜单栏小部件、导航树
* [数据小部件](data-widgets) 对于构建Mendix表单至关重要；通过这些小部件您可以查看和编辑应用程序中的数据
    * 示例：数据视图，数据网格
* [布局小部件](layout-widgets) 形成接口的主干，通常用于布局
    * 示例：布局容器，标题
* [常见小部件](common-widgets) 在任何页面、 布局或代码片段
    * 示例：标签，图像
* [容器部件](container-widgets) 可以包含其他部件
    * 示例：表、组框
* [输入小部件](input-widgets) 可以显示和编辑属性和关联的值
    * 示例：文本框，日期选择器
* [文件小部件](file-widgets) 允许您处理文件, 包括存储在文件中的图像
* [按钮部件](button-widgets) 是触发动作的按钮
    * 示例：保存按钮，微流程按钮
* [报告小部件](report-widgets) 累计数据并以表格或图表的形式显示
    * 实例：基本报告、报告图表
* [图表小部件](chart-widgets) 以图形方式表示使用各种图表类型的数据
    * 示例：线形图，条形图，热图
* 附加组件可以从 [Mendix Marketplace](https://marketplace.mendix.com/) 下载，或者由您自己使用 JavaScript 创建。
