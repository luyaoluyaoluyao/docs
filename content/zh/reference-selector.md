参考选择器是 [输入小部件](input-widgets) ，可用来显示和编辑 [关联](associations)。

{{% alert type="info" %}}

![](attachments/pages/reference-selector.png) 此参考选择器允许您为您的订单线选择商品。 ![](attachments/pages/reference-selector-domain-model.png) 要使用参考选择器，您需要一个域模型中的多对一个关联。

{{% /报警 %}}

参考选择器类似于 [报告参数](report-parameter) ，但允许用户从列表中选择要填写关联的对象，而不是枚举中列出的项目。 选择器要求您选择一个父对象共享引用的实体。 它还将要求您从您想要在选择器中显示的引用实体中选择哪个属性。

{{% alert type="success" %}}

如果您只想显示 __ 信息，您也可以使用文本框。 这增加了您可以跟随多个关联步骤的优势。

{{% /报警 %}}

## 常规属性

### 选择使用

参考选择器允许最终用户通过使用下拉页面或弹出页面选择对象。 如果您选择使用一个页面， 下拉功能将被窗口小部件右侧的按钮替换，该按钮将打开一个选择弹出窗口。

| 值    | 描述           |
| ---- | ------------ |
| 页    | 使用弹出页面选择参考值。 |
| 下拉列表 | 使用下拉菜单选择参考。  |

{{% alert type="success" %}}

使用下拉菜单选择的优点是它效率很高；不需要打开页面。 使用页面选择的优点是最终用户可以搜索对象。 如果有很多对象(例如20多个对象)，最好使用一个页面来选择。

{{% /报警 %}}

_默认值：_ 下拉

### 空选项标题

此属性代表显示给用户的下拉引用选择器中空选项的标题。 如果使用属性设置为页面选项则设置字幕不可用。 这是一个可翻译的文本。 欲了解更多详情，请参阅 [可翻译文本](translatable-texts)。

{{% alert type="info" %}}

填写空选项的标题增加了应用程序的用户体验。 它还帮助屏幕阅读器用户轻松地操作应用程序。

{{% /报警 %}}

{{% alert type="info" %}}

空选项可从 Mendix 7.2.0获取。

{{% /报警 %}}

### 选择页面

选择页面属性决定在使用选择页面按钮时打开哪个页面。 此页面可以用来从所有可能的对象列表中选择关联的对象。 此页面应包含一个数据网格、模板网格或与输入参考集选择器连接到同一个实体的列表视图。

查看 [打开页面](opening-pages)。 请注意禁止打开内容中的选择页面。

{{% alert type="success" %}}

您可以通过右键单击小部件和选择“生成选择页面...”来生成一个新页面以显示。

{{% /报警 %}}

### 转到页面

转到页面让终端用户快速访问所选对象的更详细的概览。 此属性决定向用户显示哪个页面。 页面应包含一个与引用选择器选择的实体相同的数据视图。

### 转到页面设置

这些设置指定了如何打开页面。

查看 [打开页面](opening-pages) 了解更多详情。

### 日期格式 (仅针对类型日期和时间的属性)

日期格式决定参考选择器是否显示日期、时间、日期和时间，或关联属性的自定义变化。 日期和/或时间的格式取决于查看数据的用户的本地化。

_默认值：_ 日期

### 可选择的对象属性

可选对象类别中的属性决定了最终用户可以从中选择的对象。 有两种相互排斥的方式影响可选对象：

1.  微流程返回可选对象列表。
2.  天体清单是根据上下文机制和影响该机制的属性自动确定的。

### 微流

如果选择微流，微流将被调用来计算参考选择器将显示的对象列表。 只有在选择使用下拉菜单时才能使用微流。

{{% alert type="warning" %}}

请注意，此类别中的其他属性在使用微流程时没有任何效果！

{{% /报警 %}}

### 微流程设置

在微流程设置中，您可以指定传递到微流程的参数。

### XPath 约束

使用 XPath 约束，您可以添加手动约束来限制可以选择的对象列表。

{{% alert type="info" %}}

产品参考选择器的 XPath 约束 `[Instock = true()]` 将确保只有库存中的产品是可以选择的。

{{% /报警 %}}

如果微流被用于填充参考选择器，此属性将不会产生任何效果。

### 约束者

参考选择器可以受到一个或多个路径的限制。 这通常用于使一个参考选择器依赖另一个参考选择器。 例如，在您可以编辑订单行的页面中，产品选择器可以受到类别选择器的限制。 在选择一个类别后，产品选择器会受到此类别的限制，只显示该类别中的产品。

{{% alert type="info" %}}

![](attachments/819203/917938.png) _域名模型_

在域模型中，订单行有类别和产品的类型引用。 这种形式可以通过两个参考选择器进行编辑。 第三个协会，从产品到类别，说明这两个实体之间的关系。 这种“三角”形成域模型的一部分是可能的制约因素。

![](attachments/16713884/16844014.jpg) _表单_

页面有两个参考选择器，一个用于分类，另一个用于产品。 产品所用的路径受到构建三角形的域模型路径的制约。

![](attachments/16713884/16844013.jpg)

{{% /报警 %}}

如果微流被用于填充参考选择器，此属性将不会产生任何效果。

### 排序顺序

排序顺序指定了参考选择器中项目的显示顺序。 您可以在两个方向排序多个属性 (升序和降序)。 如果没有指定排序顺序，则显示属性上的参考选择器排序。

_默认值：_ 无排序顺序

## 数据源属性

### 属性路径

属性路径指定关联实体的属性在参考选择器中显示。 路径必须遵循一个类型引用关联，从数据视图实体开始。

{{% alert type="warning" %}}

订单客户/客户名/名称将允许终端用户从用户名列表中选择用户。

{{% /警示%}}!{% alert type="warning" %}}

请记住，即使您将属性连接到参考选择器，您实际上正在选择一个对象。 该属性只是为了提供对象的可视化。 在这方面，参考选择器不同于复选框和文本框等其他小部件。 在这些小部件中，您编辑属性的值。 在参考选择器中，您编辑一个类型引用的关联。

{{% /报警 %}}
